## Issues & Feature Requests

* If you find any **bug / issue / improvement / feature suggestion**, create a **GitHub Issue**.
* Provide proper **steps to reproduce** (if applicable), along with screenshots, logs, and environment details.
* Our team will review it and get back to you as soon as possible.

## Contribution Guidelines

### For contributors having write access:

* Always create a **separate branch** for your changes.

* Raise a **Pull Request (PR)** with a clear description of:

  * **What is changed**
  * **Why it is needed**
  * **Impact area**

* Keep PRs **small and focused**.

* Ensure code follows **project structure and standards**.

* Follow the official Odoo Git guidelines for commit messages:
  [https://www.odoo.com/documentation/19.0/contributing/development/git_guidelines.html](https://www.odoo.com/documentation/19.0/contributing/development/git_guidelines.html)

* Do not manually prefix commits with custom tags like fix, refactor, etc.

* Ensure commit messages are clear, structured, and aligned with Odoo standards.

---

## Testing

* Test your changes properly before raising a PR.
* Ensure no existing flows are broken.
* Verify logs do not contain unexpected errors.
* Mention testing steps in the PR description.

---

## Security

* Do not push sensitive credentials or API keys.
* Use environment variables or configuration parameters for secrets.
* Clearly highlight any security-related fixes in the PR.
